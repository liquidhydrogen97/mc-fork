# Contract: D1.2 -- z_clamped_gradient density function

## Objective

Implement and register a custom density function type `mc_fork:z_clamped_gradient` in the NeoForge mod, mirroring vanilla `minecraft:y_clamped_gradient` but reading the Z world coordinate instead of Y. Update the datapack file `datapack/data/mc_fork/worldgen/density_function/latitude_bias.json` to use this new type with the piecewise constants from `worldgen-plan.md` Phase 1. After this contract is accepted, the latitude climate gradient activates: temperature actually varies with z position, biome distribution shifts as the player travels north/south.

**Behavior of `ZClampedGradient`:**

The function takes four parameters: `from_z` (double), `to_z` (double), `from_value` (double), `to_value` (double). For a given world position with z coordinate `z_pos`:

```
if z_pos <= from_z: return from_value
if z_pos >= to_z:   return to_value
otherwise:          return from_value + (z_pos - from_z) * (to_value - from_value) / (to_z - from_z)
```

This must work for both positive and negative `from_z`, `to_z`, and z_pos. The function is symmetric around z=0 only if the operator's piecewise spec is built from two compositions (one for positive z, one for negative z, combined via `minecraft:add` and `minecraft:abs` or equivalent).

**Implementation pattern:** Mirror NeoForge's `YClampedGradient` source (in the official Mojang mappings: `net.minecraft.world.level.levelgen.DensityFunctions$YClampedGradient`). Use a record with a static `Codec` field. Implement `DensityFunction.SimpleFunction` or the appropriate sub-interface for stateless density functions. Register the type via `DeferredRegister<MapCodec<? extends DensityFunction>>` against `BuiltInRegistries.DENSITY_FUNCTION_TYPE`.

**Codec / JSON shape:** Match vanilla `y_clamped_gradient` conventions:

```json
{
  "type": "mc_fork:z_clamped_gradient",
  "from_z": -300000,
  "to_z": 300000,
  "from_value": 1.0,
  "to_value": -1.0
}
```

(Z=-300000 returns `from_value=1.0`, z=+300000 returns `to_value=-1.0`, intermediate z linearly interpolated, beyond clamps.)

**Datapack update — `latitude_bias.json`:** Replace current constant placeholder with a piecewise composition that implements the spec from `worldgen-plan.md` Phase 1 latitude bias formula. The simplest correct form:

```json
{
  "type": "minecraft:add",
  "argument1": {
    "type": "mc_fork:z_clamped_gradient",
    "from_z": 0,
    "to_z": 300000,
    "from_value": 0.0,
    "to_value": -0.7
  },
  "argument2": {
    "type": "mc_fork:z_clamped_gradient",
    "from_z": 0,
    "to_z": -300000,
    "from_value": 0.0,
    "to_value": -0.7
  }
}
```

This produces 0 bias at z=0, -0.7 bias at |z|=300k, monotonic between. The hard floor at -1.0 and the polar nullzone forcing at |z|>400k are NOT in scope for this contract — they belong to Phase 3 (polar nullzone biome).

## Test Plan

JUnit 5 unit tests in `ZClampedGradientTest.java`:

- `testFromValueAtMinZ`: z = from_z exactly returns from_value (e.g., from_z=0, to_z=300000, from_value=1.0, to_value=-1.0; z=0 → 1.0)
- `testToValueAtMaxZ`: z = to_z exactly returns to_value (z=300000 → -1.0)
- `testMidpoint`: z = (from_z + to_z) / 2 returns (from_value + to_value) / 2 (z=150000 → 0.0)
- `testQuarterPoint`: z = from_z + 0.25 * (to_z - from_z) returns from_value + 0.25 * (to_value - from_value)
- `testClampBelowMin`: z << from_z returns from_value (z=-1000000 → 1.0)
- `testClampAboveMax`: z >> to_z returns to_value (z=1000000 → -1.0)
- `testInvertedRange`: from_z > to_z works correctly (function still monotonic between bounds)
- `testNegativeZRange`: from_z=-300000, to_z=0 with appropriate value mapping
- `testCodecRoundTrip`: serialize a ZClampedGradient instance to JSON, deserialize, verify equality
- `testRegistrationKey`: registered type's key is `ResourceLocation.parse("mc_fork:z_clamped_gradient")`

Coverage target: ≥90% of `ZClampedGradient.java` (math is small and fully testable).

Integration verification (manual, captured in Review Log):

# Contract: D1.1 -- NeoForge mod skeleton

## Objective

Set up the mc-fork NeoForge mod source tree at `mod/` with Gradle, NeoForge MDK 1.21.1, a mod entry point, and successful `build` / `runClient` / `runServer` / `runData` Gradle tasks. Empty mod with no game logic — this contract delivers the build infrastructure into which subsequent contracts (z_clamped_gradient density function, geology stratification post-processor, kimberlite feature) will land code.

## Mode

development

Development contracts produce code artifacts: implementations, integrations, tools, libraries, and infrastructure. TDD is mandatory. Tests are written before implementation. Security review is triggered when the Security Scope section lists any sensitive surfaces.

## Specification

**Output:** complete NeoForge mod project at `mod/` directory in the repo root.

**Required files:**

- `mod/build.gradle` — Gradle build configuration targeting NeoForge for Minecraft 1.21.1, Java 21, with NeoForge userdev plugin, JaCoCo coverage plugin, and JUnit 5 (Jupiter) test dependency
- `mod/settings.gradle` — root project name `mc-fork-mod`
- `mod/gradle.properties` — properties for mod ID, NeoForge version, Minecraft version, mappings channel/version, Java version
- `mod/gradle/wrapper/gradle-wrapper.properties` — Gradle wrapper version (Gradle 8.x compatible with NeoForge 1.21.1 toolchain)
- `mod/gradle/wrapper/gradle-wrapper.jar` — Gradle wrapper jar
- `mod/gradlew` — Unix Gradle wrapper script (executable)
- `mod/gradlew.bat` — Windows Gradle wrapper script
- `mod/src/main/java/com/liquidhydrogen/mcfork/McForkMod.java` — entry point class annotated `@Mod("mc_fork")` with constructor that registers a startup INFO log line
- `mod/src/main/resources/META-INF/neoforge.mods.toml` — mod metadata: modId `mc_fork`, version `0.1.0-alpha`, displayName `mc-fork`, dependencies on neoforge and minecraft
- `mod/src/main/resources/pack.mcmeta` — pack format 48 (1.21.1)
- `mod/src/test/java/com/liquidhydrogen/mcfork/SmokeTest.java` — minimal JUnit 5 test that asserts the mod ID constant and verifies logger initialization (proves test infrastructure works)

**Mod metadata:**

- modId: `mc_fork`

- license: `MIT`

**Versions (pin to current NeoForge 1.21.1 release):**

- NeoForge: latest `21.1.x` release (verify at https://projects.neoforged.net/neoforged/neoforge)

- Gradle: 8.8+ (compatible with NeoForge MDK 1.21.1)
- JUnit Jupiter: 5.10.x

**Use the official NeoForge MDK template** as a starting point: https://github.com/neoforged/MDK/tree/1.21.1 — copy the canonical structure, replace placeholders, add JaCoCo + JUnit 5 setup. Do not invent custom build conventions.

This contract delivers infrastructure, not game logic. Test strategy:

## CI Gates

Run from repo root or from `mod/`:

- `./gradlew build` — must exit 0
- `./gradlew test` — must exit 0; smoke test passes
- `./gradlew jacocoTestReport` — must exit 0; report generated
- `./gradlew runData` — must exit 0; data generation harness initializes
- `./gradlew runClient --no-daemon` (interactive) — launches Minecraft client with `mc_fork` mod loaded; log contains `[mc_fork]` startup line. Verification: capture first 50 lines of log post-startup, confirm presence of mod load message.
- `./gradlew runServer --no-daemon` (interactive) — launches dedicated server with mod loaded; same log assertion.

Linter: NeoForge MDK does not ship a linter by default. Skipping for this contract; future contracts can add Checkstyle or Spotless if desired.

## Security Scope

No security-sensitive surfaces. This is empty mod scaffolding. No auth, no user input parsing, no external network calls, no data persistence beyond what NeoForge MDK provides by default. Security review not required.

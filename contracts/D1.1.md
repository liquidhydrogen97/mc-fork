# Contract: D1.1 -- NeoForge mod skeleton

## Objective

Set up the mc-fork NeoForge mod source tree at `mod/` with Gradle, NeoForge MDK 1.21.1, a mod entry point, and successful `build` / `runClient` / `runServer` / `runData` Gradle tasks. Empty mod with no game logic ŌĆö this contract delivers the build infrastructure into which subsequent contracts (z_clamped_gradient density function, geology stratification post-processor, kimberlite feature) will land code.

## Mode

development

Development contracts produce code artifacts: implementations, integrations, tools, libraries, and infrastructure. TDD is mandatory. Tests are written before implementation. Security review is triggered when the Security Scope section lists any sensitive surfaces.

## Specification

**Output:** complete NeoForge mod project at `mod/` directory in the repo root.

**Required files:**

- `mod/build.gradle` ŌĆö Gradle build configuration targeting NeoForge for Minecraft 1.21.1, Java 21, with NeoForge userdev plugin, JaCoCo coverage plugin, and JUnit 5 (Jupiter) test dependency

## Test Plan

This contract delivers infrastructure, not game logic. Test strategy:

- **JUnit 5 (Jupiter)** wired in `mod/build.gradle` with `testImplementation` for `org.junit.jupiter:junit-jupiter:5.10.x`.
- **JaCoCo** plugin applied; `jacocoTestReport` task produces HTML at `build/reports/jacoco/test/html/index.html`.
- **One smoke test** at `mod/src/test/java/com/liquidhydrogen/mcfork/SmokeTest.java`:
  - Asserts `McForkMod.MOD_ID` equals `"mc_fork"`
  - Asserts `McForkMod.LOGGER` is non-null
- Coverage target this contract: Ōēź80% of `McForkMod` class (trivially achieved given the smoke test exercises the constants).
- No integration tests yet ŌĆö NeoForge GameTest harness is wired but empty for now (added when contracts touch in-game behavior).

## CI Gates

Run from repo root or from `mod/`:

- `./gradlew build` ŌĆö must exit 0
- `./gradlew test` ŌĆö must exit 0; smoke test passes
- `./gradlew jacocoTestReport` ŌĆö must exit 0; report generated
- `./gradlew runData` ŌĆö must exit 0; data generation harness initializes
- `./gradlew runClient --no-daemon` (interactive) ŌĆö launches Minecraft client with `mc_fork` mod loaded; log contains `[mc_fork]` startup line. Verification: capture first 50 lines of log post-startup, confirm presence of mod load message.
- `./gradlew runServer --no-daemon` (interactive) ŌĆö launches dedicated server with mod loaded; same log assertion.

Linter: NeoForge MDK does not ship a linter by default. Skipping for this contract; future contracts can add Checkstyle or Spotless if desired.

## Security Scope

No security-sensitive surfaces. This is empty mod scaffolding. No auth, no user input parsing, no external network calls, no data persistence beyond what NeoForge MDK provides by default. Security review not required.

## Parallelization Strategy

Teammate executing this contract MUST dispatch subagents for the independent subtask groups below. Sequential execution of these groups is a planning failure.

- **Group A (parallel via subagent):** Gradle build files (`build.gradle`, `settings.gradle`, `gradle.properties`)
- **Group B (parallel via subagent):** Gradle wrapper files (`gradlew`, `gradlew.bat`, `gradle/wrapper/`)
- **Group C (parallel via subagent):** Java source files (`McForkMod.java`, `SmokeTest.java`)
- **Group D (parallel via subagent):** Resource files (`neoforge.mods.toml`, `pack.mcmeta`)
- **Group E (sequential, depends on A-D):** Run build verification (`./gradlew build`, `./gradlew test`, `./gradlew runData`), capture results, iterate on errors
